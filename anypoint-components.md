# Object Store
MuleSoft store data for later (for short periods) like caching

# Anypoint Exchange
Internal store (like google play), catalog of APIs/Connectors/Apps/Templates/etc
Fosters re-use
Also works for documentation, could have a private portal ( alternative, to API Experience Hub)

# Anypoint CLI
Command line tool, control anypoint platform without UI.
Automates deployments, manages envs, apps, repeatable operations, for DevOPS.

# Runtime Manager
Most used features, cloud-based UI, to deploy and manage mule apps
irrespective of the deployment, (local on-prem or cloudhub) can manage the apps
Also monitors, has scaling related features.

# Anypoint MQ
Cloud-based message (mulesoft hosted), exchange of async messages in reliable manner.
Queue, Exchange (like a JMS Topic)
Has advanced error handling (using Dead Letter Queue) ( retry for reliability )
Event driven architecture. EDA
